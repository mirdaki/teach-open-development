Learn about Open Source development

no coding required

Introduction

Open source development is a very powerful way to work on projects. It's been used to create awesome things like Linux, Firefox, and Git where hundreds or thousands of people have worked together all across the globe. It's all about open collaboration with passionate people.

In this activity you will learn about the steps to collaborate on open source projects. We'll be making a story for this exercise, so no need to code! The open source philosophy applies to many fields after all.

Before you start - 5'


Have a story outline

You can fork the GitHub repository listed below for an story outline or create your own! Keep in mind it's an outline, and should not detail very much out. That's what the activity is for.

After choosing a story, put it somewhere were everyone can edit. GitHub is great for this, but Google Docs works well for this activity too. You can also use paper and pencil, just be sure that everyone can share their work.

Story Outline

Getting Started - 10'

The first step in contributing to an open source project is finding a project you're interested in. For this activity we'll be using the story, but finding a project is as easy as a Google search. Type in any topic of interest, virtual reality, video games, machine learning and "open source" and you're bound to find something awesome.

Once you've found a project, read up about it. See what their goals are, how it works, and how they want to receive contributions. Most open source software projects will have a README and CONTRIBUTING file for this. Many projects will also have a wiki with more information or a site to chat with others about it.

Contribute - 10'

Now that you have you know more about the project, it's time to contribute to it!

  1. Break up into groups (max of 4 people).
  2. Choose a part of the project to work on.
    • The CONTRIBUTING file should point you to a list of things to do.
  3. Work with your group to make your section awesome!
    • Don't worry about keeping everything completely cohesive. That'll come in the next section.
  4. Submit your work once it's done!

Feedback - 15'

Great job getting here. Now everyone will look over the submissions and give constructive feedback. Feedback is how projects keep up their quality and make sure everything works together. It can be a little scary, but remember, everyone just wants to make the project better.

You can can either review a submission as a group or go over all of them together. Once feedback is given get with your original groups and make any improvements you might need. Once your done, submit again. This cycle might continue a few times.

Finished - 5'

Congratulations! It's time to read through your finished story. Did it end up how you expected? Some open source projects seem to grow into something else entirely, but with some coordination you can keep it awesome. If it's not already public, share your story with the world. Let other people enjoy and learn from it. Who knows, maybe you'll get a contribution from someone else!

What you will learn

Going through this activity will help you get a grasp of what it's like contributing to an open source project. You'll work with others, give feedback, and come out with a unique story.
